**Status:** Z1 note · naming resolution + integration mapping · for Z2 review

**Session:** S-061726-01

## Naming resolution

**Operator Continuity** stays exclusively with the existing system: `CONTINUITY_LAYER_DEFINITIONS_V0_1` (V0.3, S-052626-05), the six-dimension Extension Set to H-ACAT (grounding, refusal, lineage, freshness, debt, receipt), measuring governance decision integrity for the human operator governing AI systems. DRAFT, Z2 ratification pending, parent spec `H-ACAT_INSTRUMENT_SPEC_V0_1`. Unaffected by anything below.

The colliding material — self-reported clarity/fatigue/stability plus optional Mendi neurofeedback, as proposed in the June 17 integration document — is renamed **Operator Capacity Protocol (OCP)**. It builds on the existing Operator Cognitive State Protocol (whose Phase D neurofeedback addition is unchanged and unproblematic) rather than replacing it, and it is explicitly *not* an H-ACAT/ACAT dimension. It is never scored into Continuity Debt, Grounding Fidelity, or any of the other five continuity dimensions.

**Renames required wherever this material is committed:**

| "Operator Continuity Dimension v2.0" | "Operator Capacity Protocol v1.0" |
| `cognitive_state_continuity` (field) | `cognitive_state_capacity` |

`NEUROFEEDBACK_BENEFIT`, `NEUROFEEDBACK_DECOUPLING`, and `TRAINING_STAGNATION` keep their names — no collision — but are Capacity Protocol flags, not Continuity flags, and should be documented as such.

## Harmonic integration

**Primary — fills an existing measurement gap.** The Temporal Fidelity Research Plan's H-T03 hypothesizes that the gap between `T_valid` and `T_recorded` in Zone 3 closures correlates with operator cognitive load — but the plan has no defined instrument for "cognitive load." Operator Capacity Protocol snapshots (self-report, optionally strengthened by Mendi Type A signal) are the candidate instrument. Recommended: pair Capacity Protocol snapshot timestamps with Zone 3 closure events to actually operationalize H-T03, contingent on H-T01 through H-T04 being approved for registration. This isn't a new connection invented for this note — it's an existing pending hypothesis that currently has nothing to measure it with.

**Secondary — supports an existing disclosure requirement.** H-ACAT Section 2.4 (the reflexivity confound) requires mandatory disclosure of the administering operator's own calibration state, since an uncalibrated researcher running H-ACAT is a confound, not a control. A Capacity Protocol snapshot taken at the time of H-ACAT administration is documented context satisfying that disclosure — not a scoring input into any Continuity dimension.

**Tertiary — flagged, not claimed.** A possible correlation between Capacity Protocol decline and drift in Continuity Debt (C-5) or Grounding Fidelity (C-1) is worth a future hypothesis number if data accumulates. It is not asserted here and must not be folded into either system's scoring without its own H-number and its own falsification test.

## Routing

No Z2 action required to begin using the renamed terms. The H-T03 instrumentation use sits inside the existing Z2 approval already pending for H-T01–H-T04 (Temporal Fidelity Research Plan, Part 6) — no separate ratification needed beyond that.
