### IC-019: Make OAuth Reauth Carried Forward Incorrectly While Replacement Pipeline Existed (Apr 7, 2026)

- **Detected by:** Night (repeated flag across 2+ sessions)
- **Error:** The CI listed "Make OAuth reauth — all 5 runners" as a CRITICAL-ESCALATED Zone 3 item and we continued to surface it at every session open for 8+ consecutive sessions. The replacement pipeline (GitHub Actions + n8n) was designed, built, and logged in #wgs-sync as early as April 5 (S-040526-ZETA). The decision to cancel Make.com at Gate 2 was an approved standing decision. The CI Integration Registry still listed Make.com as "ACTIVE — OAuth expired, reauth needed" without reflecting the exit plan. The OAuth reauth was therefore a dead item — we were asking Night to fix a system we had already decided to abandon — and we kept asking for 8+ sessions after the exit plan was logged.
- **Category:** PROCESS — CI not updated to reflect approved decision; dead task carried forward
- **Severity:** High — consumed Zone 3 attention across multiple sessions; caused Night to question whether the process was tracking approved decisions
- **Root cause:** Two-part failure:
  1. When the Make exit plan was approved (S-040526-ZETA, April 5), the CI was not updated to remove Make OAuth from the CRITICAL-ESCALATED queue and replace it with the migration deployment steps.
  2. Session-to-session, the CRITICAL-ESCALATED checklist was read from CI state rather than verified against the most recent approved decisions in #wgs-sync. This created a gap where #wgs-sync held the truth and CI held a stale version.
- **Corrected:** CI v4.2 (this session, April 7, OR&D Day 28):
  - Make OAuth reauth removed from CRITICAL-ESCALATED table entirely
  - Integration Registry updated: Make.com status changed from "ACTIVE — OAuth expired, reauth needed" to "ACTIVE — SUNSET AT GATE 2 · Do not reauth · Migration in progress"
  - n8n status updated: "ACTIVE — import pending · Replace Make runners" (promoted from stub)
  - GitHub Actions added to Integration Registry as new entry: "ACTIVE — acat_pipeline_trigger.yml built · deploy to .github/workflows/ · replaces Make runners"
  - New PIPELINE MIGRATION section added to CI under Gate Framework with 4-step migration sequence and Zone 3 deployment actions
  - Moving Forward entry added: Make.com exit plan is the active pipeline direction; never surface Make OAuth reauth again
- **Prevention:** Standing rule added to CI: When an exit or migration decision is approved, the replaced system must be marked SUNSET in the Integration Registry in the same session. Exit decisions do not wait for the next CI version bump.
